  ## What You're Building                                                       
                                                                                
  vibe.fm is a music intelligence mobile app. It does what Shazam does, then    
  goes deeper: it extracts a full sonic fingerprint for any identified track,   
  classifies it into a semantic archetype, and matches users with others who    
  share the same acoustic taste.                                                
                                                                                
  Four tabs. One clean data pipeline. No backend server — all API calls happen  
  client-side or from a pre-baked JSON file.                                    
                                                                                
  ---                                                                           
                                                                                
  ## Tech Stack                                                                 
                                                                                
  **Mobile:** Expo SDK 56, React Native, Expo Managed Workflow (CNG). Targets

  ---                                                                           
                                                                                
  ## Data Pipeline (How Features Are Generated)                                 
                                                                                
  There is no backend server. There is no audio analysis library. Features come 
   from APIs.                                                                   
                                                                                
  **Live scan flow (Capture tab):**                                             
  1. Record 5s of audio via `@siteed/expo-audio-studio` — outputs raw PCM, no   
  WAV header                                                                    
  2. Base64 encode the raw PCM bytes and POST to Shazam `/songs/v2/detect`      
     - Audio MUST be: 44100Hz, mono (1 channel), signed 16-bit PCM              
  little-endian                                                                 
     - Payload < 500KB, sent as plain text body                                 
  3. Shazam returns `track.title` + `track.subtitle` (artist)                   
  4. GET `https://api.freqblog.com/lookup?track={title}&artist={artist}` with   
  `X-API-Key` header                                                            
     - Returns: `bpm`, `key`, `energy`, `valence`, `danceability`,              
  `acousticness`, `instrumentalness`, `speechiness`, `mood`, `genre`            
  5. POST to OpenRouter with the features JSON → Gemini 2.5 Flash generates a   
  2-3 sentence qualitative description AND classifies into a sonic archetype    
  from `sonic_profiles.json`                                                    
     - Use `response_format: { type: "json_object" }` — mandatory               
     - Disable thinkin                                                          
  ──── (143 lines hidden) ───────────────────────────────────────────────────── 
  — this breaks byte alignment. Strip the bytes before encoding.

  ## OpenRouter / Gemini Call                                                   
                                                                                
  Model: `google/gemini-2.5-flash`                                              
                                                                                
  ```javascript                                                                 
  const response = await fetch("https://openrouter.ai/api/v1/chat/completions", 
   {                                                                            
    method: "POST",                                                             
    headers: {                                                                  
      "Authorization": `Bearer ${OPENROUTER_API_KEY}`,                          
      "Content-Type": "application/json"                                        
    },                                                                          
    body: JSON.stringify({                                                      
      model: "google/gemini-2.5-flash",                                         
      response_format: { type: "json_object" },                                 
      reasoning: { exclude: true },  // disable thinking for speed              
      messages: [{                                                              
        role: "user",                                                           
        content: `Given this track's acoustic data:                             
  ${JSON.stringify(features)},                                                  
        and these sonic archetype profiles: ${JSON.stringify(sonicProfiles)},   
        return ONLY a JSON object with:                                         
        {                                                                       
          "qualitativeDescription": "2-3 sentences describing the sonic texture 
   and emotional character",                                                    
          "sonicProfile": { "name": "matched archetype name", "tags": ["tag1",  
  "tag2"] }                                                                     
        }`                                                                      
      }]                                                                        
    })                                                                          
  });                                                                           
  ```
